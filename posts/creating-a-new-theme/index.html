Creating Personal Website Using Hugo

Aug 22, 2020 in GITHUB • HUGO • PERSONAL WEBSITE
hugo github personal website
6 min read

Introduction to Hugo

Hugo is one of the most popular open-source static site generators. With the ability of generating most websites within seconds, it provides the users with the world’s fastest framework for building personal websites.

The installation and deployment of Hugo are simple and efficient. With just 7 simple steps, we can easily use Hugo to build our own personal websites. Before introducing the 7 steps, the reader needs to get a Github account. The following instructions are based on the assumption that the users already have their personal Github accounts.

Github account is important, since it offers a static web hosting service called Github Pages. If you don’t have a Github account, please go to the Github website and create one.

Our goal in this blog is to start a personal website on Github Pages using Hugo.

Step 1: Install Homebrew and Go

As a mac user, all of the following instructions will be based on macOS environment.

Before installing Hugo on macOS, Homebrew and Go are needed. Homebrew is an open-source software package manager that makes it easier to install software on macOS and Linux, and Go is an open source programming language that makes it easy to build simple, reliable, and efficient software. These are two important tools to install Hugo. If you already have them on your computer, you can skip to Step 2.

To install Homebrew, just paste the following code in a macOS Terminal.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

After installing Homebrew, we can run the following code in the macOS Terminal to install Go.

brew install go

Step 2: Install Hugo

With the installation of Homebrew and Go, installing Hugo can be a simple task. Just type the following command in the Terminal.

brew install hugo

If you are curious whether you have already installed Hugo, just check with the following code.

hugo version

If something like this appears, the installation is successful.

Hugo Static Site Generator v0.74.3/extended darwin/amd64 BuildDate: unknown

Step 3: Set up a workplace

In this step, we need to build a directory to store our website, and the directory is based on our choice. For example, if we’d like to construct the website on our desktop, we can run the following code in the Terminal.

cd ~/Desktop
hugo new site mysite

If we can see the following result in the Terminal, the folder called mysite will appear on our desktop, and everything about our personal website will be stored in it.

Congratulations! Your new Hugo site is created in ~/Desktop/mysite.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

The structure of our newly created mysite directory will look like this:

~/Desktop/mysite
└───archetypes
│   └───default.md
└───config.toml
└───content
└───data
└───layouts
└───static
└───themes

The structure looks complicated, but a good thing is that we only need to focus on the themes folder at present. In the next step, we will choose the theme of our own personal website and use it to fill the themes folder.

Step 4: Choose a theme

It is exciting to know that Hugo has already prepared for us with plenty of pre-made, open-sourced themes which are quite abundant and can satisfy most of our needs for building a personal website. With all of the themes provided at the official website Hugo themes, what we should do first is selecting one that is appropriate for our usage. As for me, I chose hyde-hyde theme for my website.

After choosing the right theme, we can run the following command in our Terminal

cd mysite
cd themes
git clone https://github.com/htr3n/hyde-hyde.git

The github link can be obtained at the Installation part of hyde-hyde theme. The first two commands are aimed to go to the themes folder of our repository to store all of the items into it. Based on this objective, a simpler way is to download everything from the Github link to the themes folder.

Step 5: Create my website

Now we have built the framework of our personal website, we can then set up some configurations or customizations. These manipulations can be done by adding or modifying stuffs in config.toml and folders other than themes. Or more simply, we can copy everything from the folder exampleSite to our directory. During this process, just overwrite any duplications.

To see what our website looks like currently, we can test the site locally by running the following commands.

cd ..
hugo server

Step 6: Host on Github Pages

To host my website on Github Pages, I need to firstly create a new github repository.

The process of creating Github repository can be shown below.

  1. Get into the Repositories section at Github profile webpage.
  2. Click on the New button to create a new repository.
  3. Fill the Repository name space according to the baseurl in config.toml.

For the third procedure, for example, my baseurl in config.toml is https://Mingxuan-Yang.github.io, then I will fill the Repository name space with Mingxuan-Yang.github.io.

Once we have created a Github repository, there will be a link started with https, which in my example is https://github.com/Mingxuan-Yang/Mingxuan-Yang.github.io.git. Copy this link and we can turn to deploy our personal website to the Github Pages.

In the Terminal, run the following code:

hugo
cd public
git init
git add .
git remote add origin https://github.com/Mingxuan-Yang/Mingxuan-Yang.github.io.git
git commit -m “first commit”
git push origin master

We are almost done! After successfully committed, we can go to the repository, which in this case is Mingxuan-Yang.github.io, and get into the Settings page. At the Github Pages section, we can find the link to our new website.

Step 7: Update website

If you’d like to make some changes to the contents on your website, which is highly possible to happen, you can directly make changes in the website directory and then run the following code.

git add .
git commit -m "some changes"
git push --set-upstream origin master

After about 10 minutes, the changes will appear on your personal website.

Thanks for reading this tutorial. Now let’s enjoy our fantastic website!
